# 项目待办清单

## 🚀 核心 RAG 引擎 (检索准确性)
- [x] **实现混合检索 (Hybrid Search)** - *高优先级*
    - 结合向量检索与关键词检索（如 BM25/PostgreSQL 全文检索）。
    - 实现 RRF (Reciprocal Rank Fusion) 进行结果合并。
    - *目标*: 改善针对特定标识符（ID、型号）和精确匹配的检索效果。
- [ ] **添加重排序 (Re-ranking) 步骤** - *高优先级*
    - 实现两阶段检索：首先检索出 Top-50，然后通过重排序模型（如 `bge-reranker`）进行精细排序，最终返回 Top-5。
    - *目标*: 解决向量距离接近但语义不符的问题，大幅提升检索准确率。
- [ ] **上下文查询重写 (Contextual Query Rewriting)**
    - **目标**: 解决多轮对话中的指代消解、复杂逻辑拆解、术语纠错与同义词扩展，以及元数据过滤器提取等高级检索挑战。
    - **机制**: 在检索前，利用 LLM 对原始用户查询进行智能改写（Query Rewriting）或分解（Query Decomposition），生成更适合检索系统处理的查询。
    - **考量**: 引入 LLM 会增加延迟，需在“响应速度”与“检索智能”之间进行权衡。对于需要深度理解上下文和意图的场景，其提升的准确率通常能弥补延迟。
